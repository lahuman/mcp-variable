MCP basics

MCP란 무엇인가?

MCP(Model Context Protocol)는 AI 애플리케이션이 외부 시스템의 데이터와 도구를 표준 방식으로 연결해 사용할 수 있게 해주는 오픈 프로토콜입니다.

AI 애플리케이션

Claude, ChatGPT, IDE, CLI 같은 도구가 사용자의 요청을 이해합니다.

외부 도구와 데이터

파일, 데이터베이스, API, 사내 기준정보 같은 실제 근거를 제공합니다.

표준 연결 방식

각 도구마다 따로 붙이는 대신, MCP라는 공통 방식으로 연결합니다.

쉬운 비유: MCP는 AI 애플리케이션을 외부 시스템과 연결하는 표준 포트입니다. AI가 기억이나 추측에만 의존하지 않고, 필요한 도구를 호출해 확인할 수 있게 합니다.

Why it matters

왜 필요한가?

AI는 말을 잘하지만, 사내 표준 용어와 최신 기준정보를 자동으로 아는 것은 아닙니다. MCP는 AI가 정확한 출처를 가진 도구를 호출하게 만들어 이 문제를 줄입니다.

MCP 없이 물어보면

  • AI가 학습 데이터나 웹 검색을 기준으로 답할 수 있습니다.
  • 사내 도메인 사전과 다른 용어를 제안할 수 있습니다.
  • 없는 단어도 그럴듯하게 만들어낼 위험이 있습니다.

MCP 도구로 물어보면

  • 정해진 CSV 도메인 사전만 기준으로 변환합니다.
  • 정확히 있으면 변환하고, 없으면 없다고 알려줍니다.
  • 개발자가 표준명과 변수명을 빠르게 확인할 수 있습니다.

Architecture

MCP는 이렇게 연결됩니다

사용자는 AI에게 자연어로 요청하고, AI 애플리케이션은 MCP Client를 통해 MCP Server의 도구를 발견하고 호출합니다.

사용자 “이 한글명을 변수명으로 바꿔줘”
→
MCP Host AI 앱 또는 IDE가 요청을 해석
→
MCP Client 연결된 MCP Server와 통신
→
MCP Server 도구, 리소스, 프롬프트 제공
→
Tool 실행 convert_terms 호출

공식 MCP 아키텍처는 Host, Client, Server 구조와 JSON-RPC 기반 데이터 계층, stdio 또는 HTTP 기반 전송 계층으로 설명됩니다.

Project role

이 프로젝트는 무엇을 하나요?

mcp-variable은 CSV로 관리되는 도메인 사전을 읽어 한글 용어명, 물리명, 변수명을 빠르게 변환하는 로컬 MCP 서버입니다.

CSV 사전 로딩

용어명, 물리명, 도메인유형, 도메인, 데이터타입 컬럼을 읽습니다.

메모리 인덱스

용어명과 물리명을 빠르게 찾도록 속성, 단어, 도메인 맵을 만듭니다.

도구 호출

MCP Client가 convert_terms 도구를 호출합니다.

구조화 응답

변환 결과, 신뢰도, 후보, 미매칭 항목을 함께 돌려줍니다.

Conversion example

한글명·물리명·변수명을 한 번에 연결합니다

도메인 사전에 등록된 표준 용어는 전체 속성명으로 정확히 변환하고, 물리명 기반 camel case도 다시 한글 용어명으로 되돌릴 수 있습니다.

한글 용어명 물리명 변수명 lowerCamel 결과 상태
라우팅결과값 ROTNG_RSLT_VAL rotngRsltVal exact
등록일자 REG_YMD regYmd exact
API사용여부 API_USE_YN apiUseYn exact
한글 → 물리명 물리명 → 한글 camel → 한글 snake → lowerCamel

Dictionary governance

도메인 사전 연계가 주는 핵심 가치

이 도구의 목적은 단순 변환을 넘어, 표준 사전과 개발 작업 사이의 거리를 줄이는 것입니다.

기존 용어 확인

동일한 한글명이 이미 등록되어 있으면 물리명과 변수명을 바로 확인합니다.

후보와 부분 매칭

검증된 단어와 도메인 조합으로 가능한 후보를 보여주고 애매함은 표시합니다.

미등록 판단

사전에 없으면 confidence: none으로 응답해 신규 추가 요청 여부를 빠르게 판단합니다.

예: 아라처럼 사전에 없는 단어는 변환 결과를 꾸며내지 않고 unmatched: ["아라"]로 알려줍니다.

Developer impact

개발에는 어떤 도움이 되나요?

변수명은 작은 문제처럼 보이지만, 프로젝트가 커질수록 표준화와 리뷰 비용에 큰 영향을 줍니다.

표준화

팀 기준에 맞는 이름

같은 의미의 용어가 여러 이름으로 흩어지는 일을 줄입니다.

속도

반복 작업 감소

사전 검색, 약어 확인, camel 변환을 한 번의 도구 호출로 처리합니다.

협업

리뷰 기준 명확화

변수명 논쟁을 줄이고, 사전에 없는 항목은 신규 요청으로 분리합니다.

Closing

AI가 웹을 찾는 대신 사내 사전을 호출하게 만든다

MCP는 AI와 사내 시스템 사이의 표준 연결 방식이고, mcp-variable은 그 연결을 도메인 사전과 변수명 생성 업무에 적용한 예입니다.

“개발자는 이름을 고민하는 시간을 줄이고, AI는 기준 데이터에 근거한 답을 제공합니다.”
