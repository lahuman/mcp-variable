적용 업무

적용 업무: DTO 필드

DTO 필드는 업무 의미가 코드에 처음 드러나는 자리입니다. 기본 원칙은 한국어 업무 용어를 모아서 `lowerCamel`로 변환하고, exact 결과만 바로 적용하는 것입니다.

등록일자

regYmd

confidence: exact

사용여부

useYn

confidence: exact

사용자번호

userNo

confidence: exact

적용 업무

적용 업무: API payload key

API request와 response key는 DTO와 같은 사전 기반 이름을 공유해야 합니다. 그래야 클라이언트와 서버가 서로 다른 추측을 하지 않습니다.

추측 기반

  • registeredDate, regDate, createDate가 섞임
  • API 문서와 DTO 이름이 다르게 진화
  • 리뷰에서 취향 논쟁이 반복됨

사전 기반

  • 등록일자는 regYmd로 확인
  • 사용여부는 useYn으로 확인
  • payload key와 DTO 필드가 같은 근거를 공유

적용 업무

적용 업무: DB 컬럼과 물리명

DB 컬럼과 물리명은 `snake` 출력으로 맞춥니다. 마지막 물리 토큰은 도메인 물리 토큰으로 해석되므로, 약어와 데이터 의미를 함께 확인할 수 있습니다.

적용 업무: SQL alias와 검색 조건

SQL alias와 검색 조건이 애플리케이션 이름과 어긋나면, 쿼리를 읽는 순간 의미를 다시 번역해야 합니다. 같은 한국어 용어에서 SQL과 코드 이름을 함께 뽑으면 drift가 줄어듭니다.

한 용어에서 두 표면으로

사용자번호
→
USER_NO
→
userNo

쿼리 예시

select
  u.USER_NO as USER_NO,
  u.USE_YN as USE_YN
from USER_INFO u
where u.REG_YMD >= :startRegYmd

적용 방법

적용 방법: Skill 워크플로우

MCP는 도구를 제공하고, Skill은 AI에게 그 도구를 써야 하는 순간과 판단 기준을 알려줍니다.

1. 용어 수집업무 한글명을 먼저 모음
2. bulk 입력줄 단위로 `convert_terms` 호출
3. case 선택code는 `lowerCamel`, DB는 `snake`
4. 결과 판단`confidence`, `warnings`, `candidates` 확인
5. 적용검증된 이름만 반영

적용 방법

적용 방법: convert_terms 입력 예시

변환은 자유 번역 프롬프트가 아니라 구조화된 도구 호출입니다. 여러 용어는 newline-separated bulk input으로 보냅니다.

DTO 필드명

{
  "text": "등록일자\n사용여부\n사용자번호",
  "direction": "term_to_physical",
  "outputCase": "lowerCamel"
}

DB 컬럼명

{
  "text": "등록일자\n사용여부\n사용자번호",
  "direction": "term_to_physical",
  "outputCase": "snake"
}

적용 방법

적용 방법: confidence 판단 규칙

도구가 이름을 반환해도 바로 쓰지 않습니다. `confidence`, `unmatched`, `warnings`, `candidates`, `reverseCheck`를 보고 적용 여부를 결정합니다.

exact사전에 등록된 정확한 매핑입니다. 바로 적용할 수 있습니다.
composed검증된 단어와 도메인 조합입니다. warning과 candidate를 확인한 뒤 적용합니다.
partial일부만 찾았습니다. 최종 이름을 조용히 확정하지 않습니다.
none사전에 없습니다. 웹 검색이나 일반 번역으로 대체하지 않습니다.

적용 효과

적용 효과: 추측에서 표준으로

눈에 보이는 변화는 naming decision이 흩어진 개인 판단에서 하나의 반복 가능한 workflow로 이동하는 것입니다.

Before

  • AI가 웹 검색, 일반 지식, 기억으로 이름을 제안
  • 개발자마다 약어와 도메인 해석이 다름
  • 리뷰에서 이름 변경 요청이 반복됨

After

  • 신뢰하는 CSV 사전에서 이름을 확인
  • `convert_terms` 결과와 `confidence`를 함께 검토
  • Skill이 같은 절차를 매번 재사용

마무리

변수명은 추측하지 않고 사전에 확인한다

팀이 AI에게 줄 요청도 짧아집니다. Skill을 따라 DTO 필드명으로 변환해줘 라고 말하면, AI는 사전 확인, case 선택, confidence 판단까지 같은 절차로 움직일 수 있습니다.
