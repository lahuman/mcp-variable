# SSE Index Guide Design

## Goal

Serve a public `/index.html` page from the SSE server that explains what `mcp-variable` does and how to connect to it safely.

## Security Boundary

- `/index.html` is public and does not require an API key.
- The page must not print the real API key value.
- Examples use `<api-key>` placeholders for client headers.
- Deployment documentation may show how to configure `MCP_VARIABLE_API_KEYS=variable-mcp-with-dataportal`, because that belongs to server-side setup.
- `/sse` and `/messages` remain protected by the existing API key controls when `MCP_VARIABLE_API_KEYS` is configured.

## Page Content

The page should be a single self-contained HTML document generated by the server:

- Project name and short Korean description.
- Endpoint summary for `/index.html`, `/health`, `/sse`, and `/messages`.
- MCP client configuration example using the SSE URL.
- API key usage example with `Authorization: Bearer <api-key>` and `X-API-Key: <api-key>`.
- Tool summaries for `convert_terms` and `search_terms`.
- A short sample conversion result for `등록일자 -> regYmd`.
- Operational note that the real key is configured via `MCP_VARIABLE_API_KEYS`.

## Implementation Shape

- Keep the page in `src/server_sse.ts` to avoid adding a static asset pipeline.
- Add a `sendHtml` helper with security headers and `Content-Type: text/html; charset=utf-8`.
- Serve both `/` and `/index.html` so browser users have a friendly landing path.
- Do not apply rate limits or API key auth to `/index.html`.

## Testing

Add SSE server tests that:

- `GET /index.html` returns `200 text/html`.
- The HTML includes the project name, endpoint paths, and placeholder API-key examples.
- The HTML does not contain the real configured key value.
- `GET /` returns the same guide page.

Run:

```bash
npm test -- tests/server-sse.test.ts
npm test
npm run typecheck
npm run build
```
